# Python Reference

A blazing-fast, local, keyboard-first Python reference for interview prep. Type a
keyword, get instant ranked results, click to read a one-screen code-first article.

## How it's built for speed

- **Search runs in the browser, in memory.** On load, the page fetches one compact
  index (`title` + `keywords` + `category` + `related` — no article bodies) and
  searches it with [MiniSearch](https://github.com/lucaong/minisearch): field-weighted,
  prefix, and *light fuzzy* (typos like `tread`→`thread`, `queu`→`queue` still match).
  Every keystroke is a zero-network, sub-millisecond match.
- **Article bodies load on demand** from the local FastAPI server, one small article
  at a time.
- **SQLite is the source of truth.** A build step ingests markdown → SQLite → exports
  the static search index.

This scales to thousands of articles: the index stays small (only searchable
metadata), and bodies are never bulk-loaded.

## Quick start

```bash
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt

# Build the DB + search index from markdown
./.venv/bin/python server/build_index.py

# Run the server
./.venv/bin/python -m uvicorn server.app:app --host 127.0.0.1 --port 8000
```

Open http://127.0.0.1:8000

> **Note:** `server/reference.db` and `static/index.json` are generated by
> `build_index.py` and are not committed — always run the build step after
> cloning (and after editing any article). The `source/` book manuscripts are
> kept private and are not part of this repo; the articles in `content/` are the
> self-contained content the app serves.

### Keyboard

- Type to search (box auto-focuses)
- `↑` / `↓` move through results, `Enter` opens
- `/` focuses the search box from anywhere
- `Esc` clears the query
- Hover a code block → **copy** button

## Adding content (the loop)

1. Add or edit a markdown file in `content/articles/`.
2. Run `./.venv/bin/python server/build_index.py`.
3. Refresh the browser.

### Article format

Each article is one markdown file with YAML frontmatter:

```markdown
---
id: lru-cache                 # unique; defaults to the filename
title: LRU Cache              # required
keywords: [lru, cache, ordereddict, doubly linked list, O(1)]
category: Data Structures
related: [ordereddict, doubly-linked-list, functools-cache]
---
# LRU Cache

...one screen of mostly Python code...
```

- **keywords** drive search — add every term you'd type to find this (synonyms,
  method names, related concepts).
- **related** ids render as clickable chips at the bottom and power concept discovery
  (e.g. `lru` surfaces the doubly-linked-list and OrderedDict articles).
- In body text, `[label](#some-id)` links to another article by its `id`.

## Layout

```
server/
  app.py            FastAPI: serves UI, /api/index, /api/article/{id}
  build_index.py    markdown -> SQLite -> static/index.json
  reference.db      SQLite (generated)
content/articles/   one markdown file per article (source content)
web/                index.html, app.js, style.css, vendor/ (offline libs)
static/index.json   exported search index (generated)
```
